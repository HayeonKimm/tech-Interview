React를 사용하시는 이유는 무엇인가요? 

과거의 정적인 페이지보다 동적인 페이지가 많이 사용되는 요즘, 잦은 페이지 이동은 사용자에게 불편함. ( 화면의 깜박임 ) 
을 가져다 주었습니다. 따라서 SPA( Single Page Application )을 지향하는 React를 사용하게 되었습니다.
 배치된 요소를 꾸며주는 페인팅 작업을 거치게 되는데 
이러한 페인팅과 리플로우가 잦을수록 성능 저하를 가져오기 때문에 React는 화면에 
표시되는 DOM과 동일한 DOM을 메모리 상에 만들고 DOM 조작이 발생하면 메모리상에 생성된 가상 돔에 모든 연산을 수행한 후
실제 DOM을 갱신하여 리플로우, 리페인트 연산을 최소화 하였습니다.
컴포넌트 기반으로 작성되어 컴포넌트를 재사용할 수 있고 생산성 향상 뿐만 아니라 코드 유지보수에 도움을 줍니다.
